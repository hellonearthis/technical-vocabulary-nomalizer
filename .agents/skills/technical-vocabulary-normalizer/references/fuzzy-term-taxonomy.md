# Fuzzy-Term Taxonomy

This document outlines the taxonomy used to classify vague, metaphorical, or overloaded terms found in architectural specifications and conversation transcripts.

## Taxonomy Categories

| Category | Description | Example Terms |
| :--- | :--- | :--- |
| **Poetic/Metaphorical** | Words borrowing physical, literary, or non-technical imagery that obscure actual implementation mechanisms. | `magic`, `secret sauce`, `blackbox`, `seamless` |
| **Private/Jargon** | Locally defined or project-specific shorthand that is not standard in industry architecture patterns. | `the blob`, `special sauce`, `custom stuff` |
| **Overloaded/Ambiguous** | Valid technical words that are so broad or generic they lose precise meaning without qualifiers. | `manager`, `handler`, `layer`, `system` |
| **Vague/Undefined** | Generic fillers indicating functionality without specifying structure, interface, or responsibility. | `helper`, `util`, `wrapper`, `stuff`, `thing` |
| **Marketing/Hype** | Promotional adjectives that represent desirable attributes but lack objective, measurable definitions. | `intelligent`, `smart`, `frictionless`, `optimal`, `efficient` |
| **Ambiguous Mechanisms** | Action verbs that indicate a relationship or process without specifying the underlying protocol or pattern. | `talks to`, `syncs with`, `stores` |
